Notes on the Internet

Vocab

  • Protocol - Rules that we set for the internet to send information and data efficiently
  • Packet - A fraction of the data that is broke into to send the information
  • Header - Information listed with the data
  • Payload - The information sent
  • Router - Checkpoints that pass information around
  • Routing table - Rules set by routers to choose the best path

Reflection

Problems We Had

First of all, everyone didn't know where our destination was, which created chaos, and some got lost during the chaos. Additionally, the routers (tables connected with 2 or 3 other tables) were given too many packets at once to sort and pass them.

Protocols We Came Up With

  • Asking "Who are you?" and "Who are your neighbors?" to map out at least some parts of the classroom.
  • Writing the starting point and the destination on the header so routers are informed enough to find the best path.
  • Listing how many packets it was sent with to tell if the person received knows they have all the packets they need.
  • Listing the order of the data that was sent so the person received can order them properly.
  • Not sending all the packets at once, so the routers won't have to deal with many packets at once.
